# SKILL: Django Unfold Admin Generator (Bukovel Architecture)

**Мета:** Генерація та підтримка модулів панелі адміністратора Django, що використовує преміальну тему `django-unfold`, з дотриманням суворих архітектурних патернів проекту (Singletons, Translations, Image Previews).

**Контекст проекту:**
- Тема: `django-unfold` (Серверний рендеринг, жодних SPA/React в адмінці).
- Переклади: `django-modeltranslation`.
- Редактор: `django-tinymce`.

---

## 🛠 Execution Steps (Алгоритм дій для ШІ):

### 1. Аналіз та Реєстрація Моделей
- Вивчи структуру моделей у цільовому додатку (наприклад, `apps/rooms/models.py`).
- Створи або онови файл `admin.py` у відповідному додатку.
- **КРИТИЧНО:** Всі класи адмінки повинні успадковуватися від `unfold.admin.ModelAdmin` (або відповідних класів Unfold), а не від стандартного `admin.ModelAdmin`.

### 2. Реалізація Спільних Патернів (Обери потрібні для моделі):

**A. Singleton Патерн (для налаштувань, єдиних розкладів тощо):**
- Якщо модель є Singleton (напр. `SiteSettings`, `SpaSchedule`), застосуй такі правила:
  - Перевизнач `has_add_permission` та `has_delete_permission`, щоб вони повертали `False`.
  - Перевизнач `changelist_view`: зроби автоматичний редірект на форму редагування першого (і єдиного) об'єкта або на сторінку створення, якщо його ще немає.

**B. Відображення Зображень (Image Previews):**
- Для моделей із зображеннями створи методи в `admin.py` за допомогою `format_html`.
- Додай ці методи в `list_display` та `readonly_fields`.
- Приклад назви методу: `get_image_preview`.

**C. Багатомовність та Текст (TinyMCE + Modeltranslation):**
- Якщо модель має перекладені поля, переконайся, що адмінка коректно працює з полями `_uk` та `_en`.
- Для великих текстових полів (Rich Text) перевизнач віджети форми на TinyMCE.

**D. Inlines:**
- Використовуй `unfold.admin.TabularInline` або `StackedInline` для пов'язаних моделей (напр., `RoomFeature` всередині `RoomType`).

### 3. Налаштування Сайдбару Unfold
- Після створення нового `ModelAdmin`, нагадай/запропонуй оновити конфігурацію `UNFOLD` у файлі `config/settings/base.py`, щоб додати нову модель у відповідну групу навігації сайдбару.

## 🚫 Жорсткі обмеження (Constraints - Ніколи не порушуй):

1. **INSTALLED_APPS Порядок:** Ніколи не змінюй порядок базових додатків. `unfold` ЗАВЖДИ має стояти перед `django.contrib.admin`.
2. **Доступи:** Використовуй тільки стандартну систему авторизації Django (`is_staff`, `is_superuser`). Заборонено використовувати `django-guardian` чи інші object-level permissions.
3. **Маршрутизація:** Заборонено створювати кастомні URL для адмінки поза стандартним паттерном Django Admin. Всі адмін-в'юхи живуть під `/admin/`.
4. **WYSIWYG:** Ніколи не підключай CKEditor або інші редактори. Суворо тільки `django-tinymce`.
5. **UI Адмінки:** Заборонено писати кастомні CSS/JS для адмінки, якщо це можна вирішити вбудованими засобами `django-unfold`.
